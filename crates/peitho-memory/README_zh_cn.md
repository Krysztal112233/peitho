# ChatGPT 记忆系统设计简述（文章摘要）

文章来源：<https://manthanguptaa.in/posts/chatgpt_memory/>

这篇文章的核心观点是：ChatGPT 的“记忆”更可能是一个分层上下文组装系统，而不是每轮都对完整历史做一次重检索（例如全量 RAG）。

## 核心结构

作者推测实际使用了四层记忆：

1. 会话元数据（临时）：设备、时区、账号活跃度等环境信号。
2. 用户长期记忆：稳定事实与偏好（显式或隐式形成）。
3. 最近会话摘要：跨会话延续所需的轻量摘要信息。
4. 当前会话窗口：当前聊天内的滑动消息窗口。

这些层再与 system/developer 指令、最新用户消息组合，形成每次请求的最终上下文。

## 工程价值

- 相比“全历史检索”，成本更低、延迟更小。
- 通过长期事实层保持个性化。
- 通过近期摘要维持跨会话连续性。
- 通过当前会话窗口保持即时推理连贯性。

它的关键取舍是：牺牲部分远期细节，换取更好的 token 效率、响应速度和系统可控性。

## 注意事项

该文是基于外部行为观察的逆向分析，不是官方实现文档。更适合作为架构假设与设计参考，而不是“确定的内部实现规范”。
